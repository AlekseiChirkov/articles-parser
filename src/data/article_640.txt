Теги: MySQL *, SQL *

Контент: Хочу рассказать о том, как создать сквозной монотонно возрастающий
 целочисленный id в MySQL (это называется sequence в других БД). Зачем
это нужно? Например, у нас есть два вида задач для сотрудников в
системе, у каждого типа задач есть свой набор атрибутов и свой
жизненный цикл (набор статусов). Для того чтобы не городить огород в
коде приложения для корректного чтения разных сущностей из БД и не
плодить колонки с NULL проще всего разнести сущности по разным
таблицам.
